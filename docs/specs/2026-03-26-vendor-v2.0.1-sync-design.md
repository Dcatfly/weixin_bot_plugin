# Vendor v2.0.1 同步设计

vendor `@tencent-weixin/openclaw-weixin` 从 1.0.2 升级到 2.0.1，本文档描述 `src/` 需要跟进的变更。

## 范围

从 vendor 变更中筛选出 3 项与 src/ 相关的改进：

1. **contextToken 磁盘持久化** — 解决重启后 contextToken 丢失、无法回复已有对话的问题
2. **发送时 contextToken 缺失容错** — 从 throw error 改为 warn + 继续发送
3. **redactBody 日志脱敏** — 截断前先脱敏 JSON 中的敏感字段

不在范围内的 vendor 变更：
- `clearStaleAccountsForUserId`（单账号场景不需要）
- `compat.ts` 宿主版本检查（框架特有）
- `resolveOutboundAccountId` 多账号路由（框架特有）
- `triggerWeixinChannelReload` 配置热加载（框架特有）
- `blockStreaming` / `streaming` 配置（框架特有）
- CLI 子命令（框架特有）
- import 路径变更、二维码文案调整

## 设计决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| contextToken Map key 结构 | 保持 chatId（不改为 accountId:userId） | src/ 单进程单账号，多账号 key 是过度设计 |
| clearStaleAccountsForUserId | 不跟进 | src/ 的 saveWeixinAccount 是同 accountId 覆盖写，不会产生同 userId 多 accountId |
| contextToken 缺失发送策略 | warn + 继续 | 加了持久化后触发概率低，但万一触发降级发送优于报错 |

## 改动详情

### 1. contextToken 持久化 — `src/messaging/inbound.ts`

**新增模块级状态：**
- `_stateDir: string` — 通过 `setContextTokenStateDir(dir)` 设置，与 sync-buf.ts 模式一致
- `_persistAccountId: string | undefined` — 通过 `restoreContextTokens(accountId)` 设置

**存储位置：** `{stateDir}/accounts/{accountId}.context-tokens.json`

**文件格式：**
```json
{"chatId1":"tokenValue1","chatId2":"tokenValue2"}
```

**新增导出函数：**
- `setContextTokenStateDir(dir: string)` — 模块级 stateDir setter
- `restoreContextTokens(accountId: string)` — 从磁盘恢复到内存 Map，设置 `_persistAccountId`
- `clearContextTokensForAccount(accountId: string)` — 清空内存 Map + 删除磁盘文件

**修改现有函数：**
- `setContextToken(chatId, token)` — 签名不变，Map.set 后调用内部 `persistContextTokens()`

**新增内部函数：**
- `resolveContextTokenFilePath(accountId: string)` — 计算文件路径
- `persistContextTokens()` — 整个 Map JSON.stringify 写盘；`_persistAccountId` 未设置时 no-op。注：单账号模式下 Map 中所有条目均属于当前账号，因此直接序列化整个 Map。

### 2. Client 集成 — `src/client.ts`

**构造函数：**
- 新增 `setContextTokenStateDir(stateDir)` 调用

**`start()` 方法：**
- 在 `startPollLoop()` 之前调用 `restoreContextTokens(accountId)`
- `onMessage` 回调中移除 `setContextToken(msg.chatId, msg.raw.context_token!)` — poll-loop 已处理，避免持久化后双写

**`sendText()` / `sendMedia()` 方法：**
- 移除 contextToken 缺失时的 throw，改为 warn + 将 undefined 传递给下层发送函数

**`logout()` 方法：**
- 在 `removeWeixinAccount(accountId)` 之前调用 `clearContextTokensForAccount(accountId)`

### 3. 防御性清理 — `src/auth/accounts.ts`

**`removeWeixinAccount()` 函数：**
- 新增删除 `{stateDir}/accounts/{accountId}.context-tokens.json`
- 与现有的凭证文件、sync buf 删除同级

### 4. 发送容错 — `src/messaging/send.ts`

4 个函数统一修改：

| 函数 | 当前行为 | 修改后 |
|------|---------|--------|
| `sendMessageWeixin` | `logger.error` + `throw Error` | `logger.warn` (继续执行) |
| `sendImageMessageWeixin` | `logger.error` + `throw Error` | `logger.warn` (继续执行) |
| `sendVideoMessageWeixin` | `logger.error` + `throw Error` | `logger.warn` (继续执行) |
| `sendFileMessageWeixin` | `logger.error` + `throw Error` | `logger.warn` (继续执行) |

更新 `sendMessageWeixin` 的 JSDoc：移除 "contextToken is required" 描述。

### 5. 日志脱敏 — `src/util/redact.ts`

**`redactBody()` 函数：**

在截断前增加敏感字段正则替换：

```typescript
const redacted = body.replace(
  /"(context_token|bot_token|token|authorization|Authorization)"\s*:\s*"[^"]*"/g,
  '"$1":"<redacted>"',
);
```

然后对 `redacted` 执行现有的截断逻辑。
